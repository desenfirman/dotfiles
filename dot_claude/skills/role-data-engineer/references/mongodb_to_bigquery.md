# MongoDB → BigQuery Pipeline Guide

**Always use the YAML-driven approach.** Never generate a standalone Python DAG for this source type unless explicitly instructed.

## Workflow

### Step 1 — Schema Discovery (mandatory first step)

Before writing any YAML, run the schema discovery script to auto-generate `projection` and `columns_definition`. Always point `--env-file` to the `.env` at the **project/repo root** (dir containing `dags/`, `docker-compose.yml`, or `pyproject.toml`). Ask the user for the path if not obvious from context.

```bash
python .agents/skills/role-data-engineer/scripts/mongodb_source_definition_generator.py \
  --conn-id            "<conn_id>" \
  --collection         "<collection_name>" \
  --destination-table  "<table_name>" \
  --sample-size        1000 \
  --output-format      universal_parser_task \
  --load-strategy      <full_refresh|incremental|merge> \
  --filter-column      <datetime_field>   \  # required for incremental or merge
  --unique-identifier  <pk_field>         \  # required for merge
  --lookback-days      <N>                \  # optional, default 0
  --env-file           "<path/to/project/.env>"
```

Load strategy guide:
- `full_refresh` — full reload each run; no filter needed (small/static collections).
- `incremental` — delete-then-insert scoped to `[data_interval_start, data_interval_end)`. Requires `--filter-column`.
- `merge` — MERGE/upsert keyed on `--unique-identifier`. Requires `--filter-column` + `--unique-identifier`.

The script outputs a ready-to-paste YAML block (`ingest-to-gcs` + `load-to-gbq-config`). Copy it directly into `config.yml`.

### Step 2 — Scaffold `config.yml`

Use `example_mongo_to_bigquery_config.yml` as the structural template. Paste the schema-discovery output into `tasks:` under the correct `task_group`. Ensure:
- All shared values (`gcp_project_id`, `gcp_dataset_id`, `mongo_conn_id`, `gcp_conn_id`) are YAML anchors in `base_config`.
- `default_args.queue: onprem` is set (MongoDB is always on-premise).
- Tags include `source:mongo` and `destination:bigquery`.

### Step 3 — Wire the DAG

Copy `parser.py` from the example directory into `dags/lnk/domain/<domain>/<function>/`. The DAG is generated by the parser; no hand-written `dag.py` is needed.
